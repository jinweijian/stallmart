# overview 目录

本目录负责项目全局视角：项目是什么、如何启动、整体架构是什么、当前代码与历史设计之间有哪些关键差异。

## 文档列表

| 文档 | 作用 |
| --- | --- |
| [getting-started.md](getting-started.md) | 新成员和 Agent 的快速启动指南。 |
| [system-architecture.md](system-architecture.md) | 当前系统架构、模块关系、数据流和基础设施。 |

## 做什么事应参考这里

- 第一次进入项目。
- 判断小程序、后端、Docker 之间的关系。
- 修改跨端链路，例如登录、订单、店铺、商品。
- 判断历史设计是否已经落地到当前代码。
