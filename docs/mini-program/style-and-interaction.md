# 小程序视觉与交互规范

本文整理自外层设计系统、商品展示页面设计和当前小程序代码。修改页面样式前先读本文。

## 设计原则

- 移动端优先。
- 顾客端路径要短：扫码 -> 浏览 -> 加购 -> 下单 -> 查看订单。
- 摊主端操作要直接：新订单 -> 接单/拒单 -> 备餐 -> 待取餐 -> 完成。
- 关键状态必须有 loading、empty、error、disabled。
- 触控区域不小于 `44px x 44px`。

## 当前基础色

当前代码 `src/app-config/index.ts` 与 `tailwind.config.js` 使用以下基础色：

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| Primary | `#FF6B35` | 主按钮、选中态、价格强调。 |
| Secondary | `#2EC4B6` | 辅助强调。 |
| Background | `#FFF8F0` | 页面背景。 |
| Foreground | `#333333` | 正文主色。 |
| Surface | `#FFFFFF` | 卡片、弹窗。 |
| Muted | `#F5F5F5` | 占位、弱背景。 |
| Border | `#E5E5E5` | 分割线。 |

外层设计系统中还有 `#2D3436`、`#F8F9FA` 等规范。当前实现应优先保持代码已有 token，后续统一时再集中调整。

## 字体与层级

| 层级 | 建议字号 | 用途 |
| --- | --- | --- |
| H1 | 24px | 页面大标题。 |
| H2 | 20px | 模块标题。 |
| H3 | 18px | 卡片标题。 |
| Body | 16px | 正文。 |
| Body-S | 14px | 次级正文。 |
| Caption | 12px | 标签、说明。 |

约束：

- 页面内标题不要随意放大。
- 按钮文字必须可读，不允许文字溢出按钮。
- 中文文案要简洁，避免解释性长段落堆在业务页面。

## 间距与圆角

| 项 | 建议 |
| --- | --- |
| 页面左右边距 | 16px 到 24px |
| 卡片内边距 | 12px 到 16px |
| 卡片间距 | 12px |
| 按钮高度 | 40px 到 48px |
| 输入框高度 | 44px 到 48px |
| 常规圆角 | 8px |
| 卡片圆角 | 12px |
| 弹窗圆角 | 16px |

## 商品卡片

适用页面：

- `pages/customer/index`
- 后续商品详情/推荐模块

结构：

```text
商品图
商品名
价格 + 加购按钮
```

约束：

- 商品名最多两行，超出省略。
- 商品图需要默认占位。
- 售罄时必须禁用加购按钮并显示明确状态。
- 加购按钮触控区域不小于 44px，即使视觉圆形只有 32px。

## 购物车悬浮栏

适用页面：

- 店铺首页。
- 购物车相关流程。

约束：

- 固定底部时必须考虑安全区。
- 空购物车时结算按钮禁用。
- 金额与数量必须来自同一购物车状态源，避免页面重复计算不一致。

## 订单卡片

适用页面：

- 顾客订单列表。
- 摊主订单管理。

结构：

```text
订单号 + 时间 + 状态
商品摘要
确认码
操作按钮
```

约束：

- 顾客端不显示摊主操作。
- 摊主端按钮必须与订单状态机一致。
- 确认码属于敏感业务信息，不在无关页面展示。

## 风格包

历史设计定义五个风格包：

| 风格包 | 适用场景 | 主色 | 背景 |
| --- | --- | --- | --- |
| 夏威夷风 | 柠檬茶、奶茶、冷饮 | `#2ECC71` | `#FEF9E7` |
| 烧烤风 | 烧烤、煎炸、小吃 | `#E74C3C` | `#FDEDEC` |
| 市集风 | 早餐、生煎、包子 | `#F39C12` | `#FFF8E7` |
| 海洋风 | 海鲜、烤鱼 | `#3498DB` | `#EBF5FB` |
| 清新风 | 素食、沙拉、轻食 | `#27AE60` | `#F4FCF8` |

当前代码默认主题与历史风格包不完全一致。实现风格包时应先建立统一 token 映射，不要在页面里散落色值。

## Tailwind 与 SCSS

当前项目同时有 `tailwind.config.js` 和页面 `.scss`。

约束：

- 页面局部样式优先放在同目录 `.scss`。
- 跨页面设计 token 放在 `app-config` 或全局样式。
- 不混用大量 inline style。
- rpx 适配规则必须统一，不要在 Tailwind arbitrary value 和 SCSS 中各写一套。

## 交互反馈

| 场景 | 要求 |
| --- | --- |
| 加入购物车 | 数量变化要有明确反馈。 |
| 提交订单 | 按钮 loading，禁止重复提交。 |
| 接单/拒单 | 操作中禁用按钮，失败可重试。 |
| 网络错误 | Toast 或错误状态，提供重试入口。 |
| 空列表 | 显示空状态，不留白。 |

## 适配

- iPhone SE 等小屏下不能出现文字重叠。
- 底部 tabBar、悬浮栏、固定按钮必须考虑安全区。
- 列表内容不能被底部操作栏遮挡。
- 所有图片都要有默认图或错误兜底。
